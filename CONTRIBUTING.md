# Guia de Contribuição

Bem-vindo ao projeto Oficina! Este documento descreve como você pode contribuir para o desenvolvimento deste sistema de gerenciamento de oficina.

## Como Contribuir

1. **Fork o repositório**: Crie uma cópia do repositório no seu GitHub.
2. **Clone o fork**: `git clone https://github.com/seu-usuario/oficina.git`
3. **Crie uma branch**: `git checkout -b minha-feature`
4. **Faça suas mudanças**: Implemente suas melhorias ou correções.
5. **Teste suas mudanças**: Certifique-se de que tudo funciona corretamente.
6. **Commit suas mudanças**: `git commit -m "Descrição das mudanças"`
7. **Push para o fork**: `git push origin minha-feature`
8. **Abra um Pull Request**: Descreva suas mudanças e aguarde revisão.

## Diretrizes

- Siga as convenções de código do projeto.
- Escreva testes para novas funcionalidades.
- Mantenha a documentação atualizada.
- Use mensagens de commit claras e descritivas.

## Reportando Bugs

Se você encontrar um bug, abra uma issue no GitHub com os seguintes detalhes:
- Descrição do bug
- Passos para reproduzir
- Comportamento esperado
- Ambiente (sistema operacional, versão do Python/Django)

## Sugestões de Melhorias

Para sugestões de novas funcionalidades, abra uma issue descrevendo:
- O que você gostaria de ver implementado
- Por que isso seria útil
- Como você imagina que funcionaria

Obrigado por contribuir!
